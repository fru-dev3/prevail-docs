Prevail is Bun-only and ships as a single compiled binary per platform.

## Operating systems

| Platform | Status |
|---|---|
| macOS arm64 (Apple Silicon) | ✅ Native binary |
| macOS x86_64 (Intel) | ✅ Native binary |
| Linux x86_64 | ✅ Native binary |
| Linux arm64 | ✅ Native binary |
| Windows (native) | ❌ Not yet |
| Windows (WSL) | ✅ Supported |

### About Windows

There's no native Windows build yet. Bun's Windows support is preview-quality as of the 1.3.x series, and Prevail leans on Bun features (single-binary compile, `bun:sqlite`, process spawning) that need that support to be solid. Native Windows binaries will come once Bun's Windows story stabilizes. Until then, run Prevail under **WSL**, where it behaves like Linux.

## Terminal requirements

Prevail is a true-color TUI. You need:

- A terminal emulator with **UTF-8** and **true color (24-bit)**.
- `xterm-256color` or better.

Tested and known-good terminals:

- iTerm2
- kitty
- alacritty
- WezTerm
- Ghostty
- GNOME Terminal

<Aside type="tip">
If the ASCII banner or the ConfigBar chips render as boxes or garbled characters, your terminal isn't in a UTF-8 / true-color mode. Check your `TERM` (`echo $TERM` should be `xterm-256color` or similar) and your terminal's color settings.
</Aside>

## The AI CLIs

Prevail brings no models of its own — it spawns CLIs you've installed and logged into, inheriting their auth, MCP servers, and skills. You need at least one of:

| CLI | Command | Get it |
|---|---|---|
| Claude Code | `claude` | [claude.com/code](https://claude.com/code) |
| Codex | `codex` | [github.com/openai/codex](https://github.com/openai/codex) |
| Antigravity | `agy` | [github.com/google/antigravity-cli](https://github.com/google/antigravity-cli) |
| Ollama | `ollama` | [ollama.com](https://ollama.com) |

`prevail doctor` reports which it can see. The more you have installed, the bigger your [council](/council/overview/) — but a single CLI is enough to start.

## Runtime footprint

- **Binary size:** ~95 MB (self-contained — Bun runtime is baked in).
- **No runtime dependencies:** no Node, no Docker, no daemon required for normal use.
- **Disk:** your vault plus a small `~/.prevail/` for config, the session DB, and connector data.
