A single model is one voice with one set of blind spots. The **council** turns that into a panel: ask once, get answers from up to four different engines in parallel, then have a **chair** read all of them and write one decisive verdict — with the *disagreement* called out, because that's usually where your real decision lives.

## The panelists

| Panelist | CLI | Notes |
|---|---|---|
| <span class="cli-claude">Claude</span> | `claude` | Claude Code |
| <span class="cli-codex">Codex</span> | `codex` | OpenAI Codex |
| <span class="cli-antigravity">Antigravity</span> | `agy` | Google's Antigravity CLI |
| <span class="cli-ollama">Ollama</span> | `ollama` | Local models at `localhost:11434` |

Prevail auto-detects which are installed and logged in. The council uses whichever are available — you don't need all four. Run a sensitive domain on Ollama alone and nothing leaves your machine.

## What happens on `/council`

```
/council should I prepay the mortgage or invest the cash?
        │
        ├─ expand panelists (each CLI, optionally × each lens)
        │
        ├─ fan out in parallel ──▶ claude
        │                      ──▶ codex
        │                      ──▶ agy
        │                      ──▶ ollama
        │
        ├─ sanitize each reply  (prevent verdict spoofing)
        │
        ├─ chair reads all replies ──▶ synthesizes ONE verdict
        │
        ├─ divergence panel  ──▶ "where panelists disagreed"
        │
        └─ persist: sessions DB + _log/ + _journal/
```

1. **Fan out.** The question goes to every panelist *as a bare turn*, in parallel. Replies stream in as they arrive.
2. **Sanitize.** Each reply is cleaned so a panelist can't spoof the final verdict (any line that starts with `## ` is rewritten to `(panelist) ## `). See [Security](/reference/security/).
3. **Synthesize.** The chair model reads all replies and writes one verdict.
4. **Surface divergence.** A panel highlights where the panelists recommended different things or weighed risk differently.
5. **Persist.** The turn is saved to the session DB and summarized into the vault's logs and journal.

## Configuring the council

```text
/council use claude codex ollama     who sits on the council
/council model codex gpt-5.4         pin a model for one panelist
/council chair claude opus           pin who synthesizes the verdict
/council config                      show the current setup
```

By default the first panelist to answer becomes the chair; pin one with `/council chair` and it persists across sessions in your [config](/reference/configuration/).

## Add your own take first

```text
/gut I lean toward prepaying — I value the guaranteed return.
/council mortgage or invest?
```

`/gut` captures your instinct *before* the panel answers. The chair weighs it into the synthesis, so the verdict engages with your reasoning instead of ignoring it.

<Aside type="note">
Council mode multiplies cost — four panelists is four calls per turn, and `lens=all` multiplies again. Prevail previews the call count and estimated cost before firing and caps fan-out at `councilMaxCallsPerTurn` (default 16). See [the ConfigBar](/cockpit/configbar/).
</Aside>

## Calibrate it over time

The `/calibration` command lets you record decisions you're confident about, building ground truth that helps you judge which panelists and chairs serve you best. Pair it with the [canonical benchmark](/benchmarks/overview/) to grade every new model against your real questions.
